# Hugo Dataset — Labeling Methodology

## Dataset Overview

- **193 labeled PR descriptions** (106 quality, 87 slop) — see `corpus_statistics.json` for per-ecosystem counts
- **Labeling protocol:** single rater using explicit rubric below
- **Exclusion:** borderline and trivial PRs excluded from dataset

## Labeling Rubric

A PR description is labeled **QUALITY** (1) if it satisfies at least 2 of these 3 criteria:

**Criterion A — Rationale present**
The description explains WHY this change was necessary: what was broken before, what the root cause was, or what problem is being solved. "Updated auth.py" fails. "Root cause: JWT validation was using a cached key that expired 6 hours after rotation, causing 401 errors" passes.

**Criterion B — Decision context present**
The description contains at least one of: rejected alternatives, acknowledged tradeoff, or specific risk assessment. This information would be lost when the PR is merged.

**Criterion C — Reviewer guidance present**
The description tells a reviewer what to scrutinize, what changed that isn't obvious from the diff, or how to verify the fix.

A PR description is labeled **SLOP** (0) if it meets all three:
1. No rationale (cannot answer "why was this needed?")
2. Interchangeable with any other PR of the same type
3. Restates what the diff already shows, or is vaguer than the diff

## Known Limitation: Single-Rater

This dataset was labeled by a single rater using the explicit rubric above. We do not report inter-rater agreement because only one rater was available during the 72-hour hackathon window.

To quantify labeling uncertainty, we report bootstrap confidence intervals on F1 (2000 iterations) which capture sampling variability even without multiple raters.

If you want to contribute labeled PRs or run a second-rater study, see CONTRIBUTING.md.

## Exclusion Criteria

Descriptions were excluded if:
- Under 8 words (too short to evaluate)
- Primary language other than English
- Trivially generated (dependency bump, whitespace-only, auto-generated by GitHub)
- Borderline (where the labeler was uncertain — excluded, not labeled)

## Signal Coverage

The benchmark uses ECS + WhatsMissing + Alignment + Engagement (no model download required). DRIS and DSS require the sentence-transformer model (`all-MiniLM-L6-v2`, ~90MB). Running `make benchmark-full` after model download provides the complete 6-signal result.

## Anti-Gaming Validation

Run `make adversarial` to verify Hugo's resistance to 50 documented attack patterns. All 48 attack attempts score below the slop threshold. See `adversarial_results.json` in this directory.
